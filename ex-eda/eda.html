Introduction

For the purposes of DSI 36100, you should be thinking about how to use data science and computational approaches to work with and gain insight from data, and the benefits of challenges of doing so.

As such, while you should be thinking about the skills you need to develop, the primary focus is on thinking about and with data, not on the fine detail of technical skill.

Per the AT2 guide, there are a range of fantastic visualisation tools and tools to help you summarise your data. One of the most exciting new tools in this space is rawgraphs.io which I highly recommend exploring.

So, here, we’re going to walk through a limited set of approaches. I have selected these to help you try and avoid some common problems! See the instructions.Rmd for some further resources if you want to delve deeper.

We’re going to use three key packages here:

  1. summarytools and particularly the dfSummary function which gives us a summary of our dataframes
  2. ggplot2 which is a ’grammar of graphics` (the gg) which gives a very flexible way of creating visaulisations in R
  3. probably something else

Intro to tools

The summarytools package lets us create summaries of our data. The most comprehensive is dfSummary which includes:

  1. Descriptions of the variable types
  2. Some basic statistics
  3. information about missing data, etc.
#summarytools::view
#you'll see here I use ! in the select, this says 'not sent_split'. Deselecting two columns with no meaningful info in this context.
# 

trump_tweets %>%
   select(-sent_split, -id_str) %>%
    summarytools::dfSummary(
      plain.ascii  = FALSE,
      style        = "grid",
      graph.magnif = 0.75,
      sent_split = FALSE, #removing the sent_split column
      valid.col    = FALSE
    ) %>% 
  print(
    method = 'render',
    headings = FALSE,
    bootstrap.css = FALSE
  )
